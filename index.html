Today I Learned

1. git

1.1 동작 원리

  • Blob(binary large object) : Blob은 파일의 내용만을 저장하고 있다. add 명령이 실행되었을때 수정된 파일들은 hash(key)형태로 .git/object/ 경로 내에 저장된다. 이때 hash 값을 얻기 위해 Zlib을 사용하는데 반대로 value를 얻어올 때도 Zlip에 key값을 입력해서 value를 추출한다. note: 따라서 로컬만으로 버전관리가 가능함.

  • Tree: 파일 이름과 기본 속성, 디렉토리 정보를 담고있다. Tree는 commit시 생성되는 object이다. Tree는 추가된 Blob 들과 , Tree리스트를 함께 기록한다.

  • commit: commit시 생성되는 또 하나의 object이며 Tree, parent(commit), author, committer의 정보를 담고있다.

note: 이렇게 git을 통해 저장한 blob, tree, commit을 협업에 용이하게 만들것이 github와 같은 remote 저장소이다.하나의 remote 저장소를 프로젝트에 참가한 여러 개발자가 공유한다.

2. bash command or specific words

  1. ls(list): to print files in working directory without hidden files
  • a: with hidden files
  • l: line by line
  1. ~(tilde): it indicate root folder that logged in user can use
  2. pwd(print working directory)
  3. $: it indicate bash is ready to input from user
  4. touch <path/filename> : to make file
  5. mv path1/filename path2 : move file in path1 to path2
  • if user write path2 + filename2, filename is changing to filename2
  1. rm path : remove file
  • *.py: all .py extension files will be removed
  • index.*: all index. files will be removed
  1. rm -r path : remove folder (r = recursive)
  2. rm -rf path : remove folder forcibly (f= force)
  3. chmod: change modification
  4. chown: change owner
  5. a commmand && b command : b is executed after a is executed.

3. vi command:

  1. u: undo
  2. Y: copy
  3. p: paste
  4. o: open line under user’s cursor
  5. O: open line above user’s cursor
